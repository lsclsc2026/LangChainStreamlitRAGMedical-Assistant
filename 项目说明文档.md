# langchain-agent-master 项目说明文档

> 面向医疗分诊与心理咨询场景的多模块智能对话系统说明。

## ① 简介
本项目是一个基于 Streamlit、LangChain、LangGraph、Chroma 的多模块智能对话系统，支持医疗分诊对话与心理咨询 GraphRAG 两类场景。

系统已实现的核心能力包括：
- 医疗分诊流程编排：信息充足度判断、风险分级、路由知识检索、建议生成。
- 心理咨询 GraphRAG：基于指定 PDF 建图，支持 Global Search 与 Local Search。
- 知识库管理：本地文件入库、重建、增量同步、来源引用展示。
- 多会话与记忆：会话持久化、上传记忆存储、历史恢复。
- 流式交互：前端实时展示流程节点与最终回答。

## ② 功能简介
- 医疗分诊对话模块：症状采集、补充追问、严重程度评估、分层建议输出。
- 心理咨询模块：模糊度判断、全局检索、局部证据召回、支持性回答。
- RAG 检索与重排：向量召回 + 可配置本地 reranker。
- OCR 能力：支持本地 OCR 与讯飞 PDF OCR 任务接口。
- 管理员能力：知识文件上传、删除、重建接口与前端操作入口。
- 后端 API：健康检查、上传文件记忆、知识库管理等 FastAPI 接口。

## ⑥ 配置说明

### 1. API Key 配置
项目使用 OpenAI 兼容接口，至少需要以下环境变量：

- OPENAI_API_KEY
- OPENAI_BASE_URL

可在系统环境变量中配置，或在项目根目录 .env 文件中配置：

可选环境变量：
- ADMIN_TOKEN：管理员接口鉴权（知识库上传/删除/重建）。
- AGENT_USER_CITY：会话默认城市。
- AGENT_USER_ID：会话默认用户 ID。
- XFYUN_OCR_APP_ID、XFYUN_OCR_API_SECRET：覆盖讯飞 OCR 配置。

### 2. config 文件说明
- config/rag.yaml
  - chat_model_name、embedding_model_name、temperature
  - local_reranker_model_path：本地 reranker 路径（可填 snapshots 根目录）
  - local_reranker_model_id：自动下载时的模型 ID
  - local_reranker_auto_download、local_reranker_device 等

- config/chroma.yaml
  - 向量库路径、检索 top_k、切块参数等

- config/prompt.yaml
  - 各提示词文件路径映射

- config/agent.yaml
  - trace_refresh_interval_sec
  - triage_perf（轻量快判、单次生成、route_top_k 等性能参数）

- config/psych.yaml
  - 心理模块 OCR 相关配置（provider、base_url、app_id、api_secret、超时等）

### 3. rerank 模型配置建议
在 config/rag.yaml 中配置：
- local_reranker_model_path: 本地模型目录
- local_reranker_device: cpu 或 cuda
- local_reranker_local_files_only: true（离线本地加载）

若路径不存在且希望自动下载，可设置：
- local_reranker_auto_download: true

## ⑧ 优化方向
当前架构的痛点很典型：医疗分诊链路里仍有串行调用，知识库一大时检索与重排成本会上升，后端每个节点如果没有明确快慢路径，整体体感依然会慢。

比较实用且业界常见的优化方向，是把当前流程升级成“路由分层 + 混合检索 + 多级缓存”的结构：先用轻量路由器判断请求复杂度，简单问题直接走快路径（规则快判/单次生成），复杂问题再走重路径（深度检索+重排）；检索层采用关键词与向量并行召回，减少纯向量误召回；同时增加会话缓存、语义缓存与热点缓存，优先命中高频问题。这样做的价值是能同时优化平均时延和稳定性，而不是只追求某一轮回答更快。
